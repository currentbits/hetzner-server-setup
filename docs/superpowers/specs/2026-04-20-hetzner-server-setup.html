🏗 Architecture

Internet ├── :80/:443 ──▶ Hetzner FW ──▶ UFW ──▶ CrowdSec ──▶ Traefik (Coolify) │ ├── Your Apps │ ├── Postgres │ ├── Redis │ └── MinIO │ └── :22 ──▶ Hetzner FW (home IP) ──▶ UFW (home IP + Tailscale) ──▶ sshd Admin UIs (Tailscale-only): ─ Coolify dashboard :8000 ─ MinIO console :9001 Backups: Coolify volumes ──▶ rclone crypt ──▶ Hetzner Object Storage (nightly, AES-256) Monitoring: Grafana Alloy ──▶ Grafana Cloud (metrics + logs) Notifications (Discord → switchable to Slack): ─ SSH login detected ─ Reboot required (pending security update) ─ Setup complete

🔒 Security Layers

Layer Type Mechanism
Hetzner Cloud Firewall Network L3 block before traffic reaches the OS. 80/443 open, 22 home IP only.
UFW Host Second firewall layer. Logs all blocked attempts → Loki/Grafana.
CrowdSec Host Community threat intelligence. Blocks known bad IPs across thousands of servers.
SSH hardening Host Key-only auth, root login disabled, home IP + Tailscale only. Login alert via Discord.
Tailscale Network Admin UIs (Coolify :8000, MinIO :9001) not reachable without VPN.
Docker daemon App no-new-privileges, AppArmor, log rotation, resource limits.
Traefik middleware App Global security headers (HSTS, X-Frame, nosniff) + rate limiting on all routes.
auditd Host GDPR baseline: auth events, sudo, /etc/ writes, SSH key changes.
rclone crypt Data AES-256 client-side encryption before backup upload. Hetzner cannot read data.
unattended-upgrades Host Non-reboot patches auto-applied. Reboot-required → Discord notification.

⚙️ Configuration Block

All secrets and environment-specific values live at the top of setup.sh. Nothing else needs editing before running.

# ─── CONFIGURATION ──────────────────────────────────────────────────────

DEPLOY_USER="lxadmin"

# Your home public IP — find it at https://ifconfig.me
HOME_IP="x.x.x.x"

# Notification webhook
# Discord: Server Settings → Integrations → Webhooks → New Webhook → Copy URL
# Slack:   api.slack.com → Your Apps → Incoming Webhooks → Add → Copy URL
WEBHOOK_URL="https://discord.com/api/webhooks/..."
WEBHOOK_TYPE="discord"   # "discord" or "slack" — change both lines to switch

# Hetzner Object Storage
# Console → Object Storage → Buckets → Create Bucket (region: Falkenstein)
# Credentials: Object Storage → Access Keys → Generate Access Key
HETZNER_BUCKET="your-bucket-name"
HETZNER_ACCESS_KEY="..."
HETZNER_SECRET_KEY="..."
HETZNER_REGION="fsn1"   # fsn1=Falkenstein, nbg1=Nuremberg, hel1=Helsinki

# Backup encryption passphrases
# Generate: openssl rand -base64 32
RCLONE_CRYPT_PASS="..."
RCLONE_CRYPT_PASS2="..."  # Second passphrase (rclone requirement)

# Grafana Cloud
# Cloud → your stack → Connections → Add new connection → Linux node
# Copy the Alloy config: URL, user ID (numeric), and API key
GRAFANA_CLOUD_METRICS_URL="https://prometheus-prod-xx.grafana.net/api/prom/push"
GRAFANA_CLOUD_LOGS_URL="https://logs-prod-xx.grafana.net/loki/api/v1/push"
GRAFANA_CLOUD_USER_ID="123456"
# Grafana Cloud → your profile → Access Policies → Create token
GRAFANA_CLOUD_API_KEY="glc_..."

# Tailscale
# Admin Console → Settings → Keys → Generate auth key (reusable: off)
# https://login.tailscale.com/admin/settings/keys
TAILSCALE_AUTH_KEY="tskey-auth-..."

SWAP_SIZE="4G"

# ────────────────────────────────────────────────────────────────────────

🚀 Script Phases

Each phase is idempotent — safe to re-run if setup fails partway through.

01

System user + SSH hardening + login alert

  • Create lxadmin with sudo, copy root's authorized SSH keys
  • sshd_config: PermitRootLogin no, PasswordAuthentication no
  • PAM hook: fires Discord alert with timestamp + source IP on every SSH login
02

Hetzner Private Network interface config

  • Configure eth1 with static private IP (found in Hetzner Console → Server → Networking)
  • Enables free inter-server traffic when a second server is added later
03

UFW firewall + logging

  • Default: deny incoming, allow outgoing
  • Allow 22 from $HOME_IP only · Allow 80/443 from anywhere
  • UFW logging on — blocked attempts flow into Loki/Grafana
04

Tailscale

  • Install via official script, auth with $TAILSCALE_AUTH_KEY
  • UFW: allow SSH from Tailscale subnet 100.64.0.0/10
05

CrowdSec

  • Install CrowdSec agent + Traefik bouncer
  • Enroll in community blocklist — auto-blocks known bad IPs
  • Install collections: linux, traefik, ssh
06

Swap file

  • Create $SWAP_SIZE (4GB) swap at /swapfile
  • swappiness=10 — prefer RAM, use swap as safety net only
  • Persisted in /etc/fstab
07

unattended-upgrades + reboot webhook

  • Automatic-Reboot: false — never reboots without your permission
  • Systemd timer (daily): checks /var/run/reboot-required, fires Discord if set
  • Non-reboot security patches applied automatically
08

Docker + hardened daemon.json

  • Install Docker CE from official apt repo (pinned version)
  • Add lxadmin to docker group
  • daemon.json: no-new-privileges, live-restore, log rotation (10m/3 files), ulimits
  • AppArmor enabled by default on Debian — Docker uses it automatically
09

Coolify install

  • Official Coolify install script
  • Coolify manages Postgres, Redis, MinIO as Docker containers
  • Dashboard on :8000 (UFW blocks from internet — Tailscale only)
  • MinIO console on :9001 (Tailscale only)
10

Traefik global security middleware

  • Configured via Coolify's Traefik dynamic config directory
  • Security headers applied to all routes: HSTS (1yr), X-Frame-Options: DENY, X-Content-Type-Options: nosniff, Referrer-Policy, Permissions-Policy
  • Rate limiting: 100 req/s per IP, burst 200 — DDoS protection at proxy level
11

auditd — GDPR baseline

  • Install and enable auditd
  • Rules: auth events, sudo usage, writes to /etc/, SSH key changes
  • Logs forwarded to Loki via Grafana Alloy
12

rclone + encrypted backup → Hetzner Object Storage

  • Install rclone, configure rclone crypt remote over Hetzner Object Storage
  • All data AES-256 encrypted client-side before upload — Hetzner cannot read it
  • Nightly cron (02:00): Postgres dumps, Redis RDB, MinIO data, Coolify config
  • Endpoint: fsn1.your-objectstorage.com (S3-compatible)
13

Grafana Alloy → Grafana Cloud

  • Install Grafana Alloy as systemd service
  • Metrics: CPU, RAM, disk, network (node_exporter built-in)
  • Logs: /var/log/syslog, /var/log/auth.log, Docker container stdout, UFW blocked IPs
  • Recommended dashboards to import: Node Exporter Full (1860), Docker (10619), Loki Logs (13639)
14

lynis security audit

  • Install lynis, run one-time audit at end of script
  • Report saved to /root/lynis-report.txt
  • Hardening index score printed to terminal — review and act on recommendations

🔔 Notification System

A single notify() function handles both Discord and Slack. Switching providers requires changing two lines in the config block — nothing else.

notify() {
  local msg="$1"
  if [[ "$WEBHOOK_TYPE" == "slack" ]]; then
    curl -s -X POST "$WEBHOOK_URL" \
      -H "Content-Type: application/json" \
      -d "{\"text\": \"$msg\"}"
  else
    curl -s -X POST "$WEBHOOK_URL" \
      -H "Content-Type: application/json" \
      -d "{\"content\": \"$msg\"}"
  fi
}

SSH Login Alert

User, source IP, and timestamp sent immediately on every SSH session start.

Reboot Required

Daily check. Fires with pending update name and how long it's been waiting.

Setup Complete

Summary notification at end of script listing installed phases.

Switching to Slack

Change WEBHOOK_URL and WEBHOOK_TYPE="slack". Done.

🐳 What Coolify Manages After Setup

Services

Postgres, Redis, MinIO deployed as Docker containers — configured via Coolify UI.

Reverse Proxy

Traefik built into Coolify. Auto-SSL via Let's Encrypt for all deployed apps.

App Deployments

Deploy from Git, Docker image, or Docker Compose. One-click deploys.

Scheduled Backups

Coolify backs up Postgres → MinIO. Our script backs MinIO → Hetzner Object Storage.

📈 Multi-Server Expansion

Coolify supports multi-server natively. Adding a second server is straightforward:

1
Provision a new Hetzner CX42 in the same datacenter
2
Add it to the same Hetzner Private Network (free inter-server traffic)
3
Run setup.sh on the new server — same script, same config block
4
In Coolify dashboard: Servers → Add Server → SSH into new server
5
Coolify installs its agent. Server is ready for deployments immediately.

Coolify can deploy the same app across multiple servers with a load balancer. No additional orchestration tooling needed at this scale.

🔀 Approach Alternatives

Option A Chosen

Bash Script

Single idempotent script. No local tooling needed. Easy to audit, version in git, and hand to a new team member. Re-run on any new server.

Option B

Ansible Playbook

Declarative, structured, great for secrets vault. Overkill for one server — migrate to this at 3+ servers or when team grows.

Option C

cloud-init

Inject script into Hetzner server creation. Zero-touch provisioning from day 0. Best for frequent re-provisioning or auto-scaling scenarios.
